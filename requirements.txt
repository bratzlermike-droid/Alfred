# Alfred — A Voice-Controlled AI Assistant

> A self-hosted, butler-themed AI assistant with a cloud LLM backend, a native desktop app, and a mobile web interface. Speak to it, and it controls your PC, manages your calendar and email, tracks your spending, transcribes meetings, and runs autonomous research tasks — all for **$0/month** on free-tier infrastructure.

## What it does

Alfred is a personal AI assistant inspired by Batman's butler. You activate it with the wake word **"Alfred"** (or `Ctrl+Space`) and speak naturally. It responds in a British butler persona through high-quality neural text-to-speech, and can take real actions across your machine and online accounts.

It spans **three coordinated surfaces:**

| Surface | Platform | Role |
|---|---|---|
| **Server** | FastAPI on Oracle Cloud (free tier) | LLM inference, speech-to-text, text-to-speech, long-term memory |
| **Desktop app** | Windows (customtkinter + animated orb UI) | Wake-word listening, PC control, screen vision, 20+ feature modules |
| **Web app** | Any phone/browser | Quick voice access with browser-native speech APIs |

## Architecture

```mermaid
flowchart TD
    subgraph Clients
        D["🖥️ Desktop App<br/>(Windows, customtkinter)<br/>wake word · holographic orb · hotkey"]
        W["📱 Web App<br/>(phone / browser)<br/>Web Speech API"]
    end

    subgraph Server["☁️ FastAPI Server — Oracle Cloud (free tier)"]
        API["REST API<br/>(Bearer-token auth)"]
        STT["Whisper STT<br/>(speech → text)"]
        TTS["Piper TTS<br/>(text → speech)"]
        MEM[("ChromaDB<br/>long-term memory")]
        ROUTER{"Tool / intent<br/>router"}
    end

    subgraph LLM["Language Models"]
        GROQ["Groq API<br/>llama-3.3-70b<br/>(primary)"]
        OLLAMA["Ollama<br/>qwen2.5:7b<br/>(local fallback)"]
    end

    subgraph Integrations["Feature Modules"]
        PC["PC control · screen vision"]
        PROD["Email · Calendar · Finance"]
        MEDIA["Spotify · Meeting transcription"]
        AGENT["Autonomous research agent"]
    end

    D -->|voice / text| API
    W -->|voice / text| API
    API --> STT
    API --> TTS
    API --> ROUTER
    ROUTER --> GROQ
    GROQ -.fallback.-> OLLAMA
    ROUTER --> MEM
    D --> Integrations
    ROUTER --> Integrations
```

**Design highlights**

- **Hybrid LLM with graceful degradation** — Groq's hosted Llama 3.3 70B is the primary engine for speed; a local Ollama model takes over automatically if the API is unavailable, so the assistant never goes fully offline.
- **Stateful long-term memory** — facts, schedules, and past conversations are embedded and stored in ChromaDB, then retrieved as context on later requests.
- **Token-authenticated API** — every client authenticates with a Bearer token; all secrets are injected via environment variables (see [Configuration](#configuration)).
- **Modular feature system** — each capability (email, finance, vision, etc.) is an independent module with its own intent detection, making the assistant easy to extend.

## Configuration

All secrets are read from environment variables — **nothing sensitive is committed to this repo.**

1. Copy the template and fill in your own values:
   ```bash
   cp .env.example .env
   ```
2. Required keys are documented in [`.env.example`](.env.example): a shared `ALFRED_AUTH_TOKEN`, the `ALFRED_SERVER_URL`, your `GROQ_API_KEY`, and optional Email/TTS credentials.

## Quick start

### Server (Linux / cloud VM)
```bash
git clone https://github.com/YOUR_USERNAME/alfred-ai-assistant.git
cd alfred-ai-assistant
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # then edit .env with your values
uvicorn chief_server:app --host 0.0.0.0 --port 8000
```
The web interface is then served at `http://YOUR_SERVER_HOST:8000`.

### Desktop app (Windows)
```powershell
pip install -r requirements.txt
# set ALFRED_SERVER_URL and ALFRED_AUTH_TOKEN to match your server, then:
python chief_desktop.py
```
